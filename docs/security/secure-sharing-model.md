# Secure Sharing Model

A shared Rental Passport must never be a simple public link. Sharing is recipient-specific, time-limited, logged, and revocable.

## Secure Passport Invitation

1. Tenant enters the intended landlord email.
2. Rental Passport creates a unique share token.
3. Rental Passport sends a secure invitation.
4. Landlord opens the invitation.
5. The intended email is already populated.
6. Landlord creates secure access instead of casually signing up.
7. Email ownership is confirmed.
8. Landlord lands in an Applications dashboard containing the invited application.

This protects applicant information while creating a secure landlord account naturally.

## Language

Do not market landlord access as "Sign Up."

Preferred language:

- Create Secure Access
- Secure This Application
- Protect Applicant Information
- Continue Securely

## Sharing Requirements

- Unique share token
- Intended recipient email
- Magic link or login requirement
- One-time code where appropriate
- Expiry date
- Revocation
- Access logs
- Tenant visibility into views
- Auto-revocation after tenant accepts a lease elsewhere

## Document Viewing Rules

Sensitive documents must never become downloadable files by default.

Supporting document viewing, when explicitly authorized, must be:

- Authenticated
- Recipient-specific
- Time-limited
- Revocable
- Logged
- View-only
- Watermarkable
- Protected from casual sharing
- Inaccessible after expiry or tenant revocation

Landlords may download only:

- Completed rental application
- Verification certificate or verification methods page
- Signed lease documents where appropriate

ID documents, credit reports, pay stubs, employment documents, leases, bank records, and reference responses remain securely viewable online only when authorized.
