# Privacy Model

Rental Passport is privacy-first. Documents stay inside Rental Passport. Landlords receive verified information, evidence summaries, completeness, and application packages, not raw files.

## Default Sharing Model

By default, landlord access includes:

- Passport Completeness
- Section verification status
- Verification summary
- Evidence summary
- Verification freshness
- Application package
- Viewable applicant profile
- Relevant activity history

By default, landlord access excludes:

- Driver's licence
- Pay stubs
- Bank statements
- Raw credit reports
- Employment letters
- Supporting documents

## Future Document Sharing

Future document sharing must be:

- Explicitly authorized by the renter
- Scoped to specific documents
- Time-limited
- Revocable
- Audited
- Visible in renter activity history

## Permission Controls

Permissions should support:

- Expiring share links
- One-time links
- Access revocation
- View tracking
- Device tracking
- Role-based access
- Least privilege
- Consent receipts

## Compliance Controls

The privacy model must adapt to jurisdiction and suppress illegal questions, filters, and landlord-facing fields where prohibited.

## Security Expectations

The platform should design for:

- Encryption in transit
- Encryption at rest
- Audit logging
- PII minimization
- Secure storage boundaries
- RLS-backed data access
- Service-role isolation
- SOC 2 readiness
- PIPEDA and GDPR readiness
- Future ISO 27001 readiness
