# User Flows

These flows describe product direction only. They are not implemented backend workflows.

## Tenant Experience

Tenant promise: build once, verify once, apply everywhere.

1. Create Rental Passport.
2. Add profile and application information.
3. Add or connect evidence.
4. Choose verification methods.
5. Review Passport Completeness and section verification states.
6. Enter intended landlord email and send a secure invitation.
7. Track views and activity.
8. Revoke or expire access when needed.

## Landlord Experience

Landlord promise: open link, trust immediately, click deeper only when necessary.

1. Receive secure passport invitation.
2. Create Secure Access to protect applicant information.
3. Land in a simple Applications dashboard containing the invited application.
4. Open the passport cover page.
5. Understand the highest-value trust signals within about 15 seconds.
6. Click into detailed sections only when needed.
7. Review verification evidence summaries and confidence levels.
8. Review income facts and income multiple when rent is known.
9. Filter applicants only by legal criteria.
10. Download permitted summaries, verification certificates, and application packages.
11. Request additional information if needed.
12. Make their own rental decision.

## Secure Sharing Flow

1. Tenant enters landlord email.
2. Rental Passport creates a unique share token.
3. Landlord receives a secure invitation.
4. Landlord confirms email ownership.
5. Landlord creates secure access.
6. Tenant can see access history.
7. Tenant can revoke access.
8. Access expires or auto-revokes when appropriate.

## Verification Flow

1. Renter submits information or evidence.
2. Rental Passport verifies the evidence through manual review, secure contact, document review, manual provider-assisted review, or future integration.
3. The section receives a verification status.
4. Where appropriate, the section receives an authenticity confidence level.
5. The passport version is updated.
6. The activity history records what changed.
7. Landlords see verified facts and evidence summaries, not raw documents.

## Regional Application Flow

1. Platform detects jurisdiction.
2. Platform selects legal questions and supported filters.
3. Platform generates the correct regional application.
4. Verified passport data pre-fills supported fields.
5. Illegal or unsupported questions are suppressed.

## Lease Flow

1. Landlord makes their own decision.
2. Rental Passport selects a jurisdiction-specific lease template.
3. Verified passport data pre-fills the lease.
4. Parties sign electronically.
5. Executed lease is stored securely.
6. Active tenancy can continue in Rental District.

## Applicant Comparison Flow, Future

1. Landlord selects applicants.
2. Rental Passport compares factual fields only.
3. The comparison never recommends, ranks, or scores applicants.
4. Landlord remains responsible for the decision.
