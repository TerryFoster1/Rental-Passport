# Regional Compliance Strategy

Rental Passport must understand jurisdiction before collecting, displaying, filtering, or generating rental application data.

## Jurisdiction Model

The product should know:

- Country
- Province
- State
- Territory
- Region
- Municipality, future

## Compliance Responsibilities

Features, filters, leases, application questions, landlord views, and verification workflows must adapt automatically based on jurisdiction.

Illegal screening questions should never appear in jurisdictions where prohibited.

Examples of sensitive or protected criteria may include:

- Pets
- Family status
- Disability
- Marital status
- Age
- Source of income
- Smoking preference
- Other protected characteristics where applicable

## Filtering Rules

Filtering is allowed. Scoring people is not.

Supported filters must be enabled only where legal. Examples include:

- Fully Verified
- Identity Verified
- Employment Verified
- Income Verified
- Rental History Verified
- Credit Report Verified
- References Verified
- Document Complete
- Application Date
- Move-in Date
- Monthly Income
- Verified Income Multiple
- Parking Required
- Other legal search criteria

## Lease and Application Rules

The regional application engine and digital lease library must select jurisdiction-specific templates automatically.

The renter should never type the same information twice, but the platform must not auto-fill illegal questions.

## Required Legal Review

Before real use, legal counsel should review:

- Screening questions
- Landlord filters
- Credit report usage
- Income presentation
- Deposit escrow
- Lease templates
- Consent language
- Jurisdiction-specific disclosure requirements
