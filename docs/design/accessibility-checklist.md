# Accessibility and UX Checklist

Rental Passport should feel simple, trustworthy, and calm. Accessibility is part of trust.

## UX Direction

Tenant experience:

- Build once.
- Verify once.
- Apply everywhere.

Landlord experience:

- Open link.
- Trust immediately.
- Review only the sections that matter.
- Click deeper only when necessary.

## Passport Summary Philosophy

The passport is not a dashboard. It is the cover page of a professional report.

The first page should answer: "Can I confidently spend time reviewing this applicant?"

A landlord should understand the application within about 15 seconds.

The main passport should stay intentionally minimal:

- Applicant Name
- Passport ID
- Verification Date
- Passport Expiry
- Fully Verified Badge
- Passport Completeness
- Credit Verified
- Employment Verified
- Income Verified
- Rental History Verified
- References Verified
- Identity Verified
- Quick Summary
- Message Applicant
- Save Applicant
- Accept Applicant

Everything else belongs behind clickable sections.

## Design Language

Use the approved mockups as the design language. The interface should feel like Apple, Stripe, Plaid, modern banking, insurance portals, and government digital identity.

Avoid CRM software, enterprise dashboards, and complex analytics patterns.

Use:

- Cards
- Trust badges
- Verification badges
- Clear status indicators
- Minimal navigation
- Generous whitespace
- Professional document-style layouts

Avoid:

- Complex navigation
- Analytics dashboards
- Graphs
- Large data tables
- Excessive metrics
- Vague labels such as AI Score, Risk Score, or Tenant Rating

## Accessibility Checklist

- Meet WCAG AA contrast targets.
- Provide visible keyboard focus.
- Use semantic headings.
- Keep status indicators text-readable, not color-only.
- Label forms and upload controls clearly.
- Explain verification states in plain language.
- Avoid jargon around legal, credit, and identity concepts.
- Ensure mobile layouts preserve trust and readability.
- Avoid motion that obscures document or verification information.
